# VayuRoute Edge (Vayu)

VayuRoute Edge is a production-grade, edge-native autonomous fleet routing system for Mumbai's highly dynamic logistics environment. It runs entirely on a standard laptop/desktop (CPU-only) and works offline after the initial data load.

---

## User Persona

Ravi is a fleet dispatcher at a Mumbai last-mile logistics company managing 12 delivery
vehicles in the Andheri-Kurla corridor. During monsoon season he loses 2-3 hours per day
to manual rerouting around flooding and traffic. He has no predictive capability and only
reacts after congestion forms. VayuRoute solves this by predicting flood risk 45 minutes
ahead, autonomously rerouting before congestion forms, and running entirely offline on a
standard laptop with no cloud subscription or internet required during operation.

## Live-Wire Integration

- Open-Meteo API feeds real-time Mumbai rain and storm data into the STGCN feature matrix.
- No API key or registration required.
- System degrades gracefully to neutral weather values if the API is unreachable.

---

## Edge Deployment

- All ML inference runs on CPU with no cloud APIs used for decisions.
- STGCN hidden dimension: 16.
- STGCN exported as TorchScript (stgcn_edge.pt), under 5 MB.
- Average inference latency under 300 ms.
- Fully offline after initial OSM graph cache (backend/data/mumbai_graph.gpickle).

## Features

- Glassmorphism UI: dark navy theme with blurred translucent panels, neon accents, and smooth animations.
- Multi-Vehicle Simulation: two independent vehicles (V1 and V2) tracked simultaneously, each with its own route, fuel level, and AI decisions.
- Baseline vs AI Routing: toggle between classic Dijkstra baseline (static distances) and the AI mode (STGCN predictions + RiskAwareRouter).
- Future Prediction Timeline: slider to set the prediction horizon (0 m, 15 m, 30 m, 45 m). Risk heat-map and edge weights scale accordingly.
- Decision Intelligence Panel: shows flood probability, fuel level, time saved, route confidence, and a human-readable explanation for each reroute.
- Delta Comparison Panel: real-time display of AI advantage: time saved, risk avoided, and fuel efficiency.
- Event Log and Performance Panel: live streaming of system events (rain, flood, fuel drop) and latency metrics.
- Time-Freeze Visual Effect: when an event is triggered the map dims, a "Predicting network failure..." overlay appears, the vehicle pauses, then the new route animates.
- Demo Script Mode: one-click automated demo that runs the full event sequence (normal, rain, flood, fuel drop) with deterministic timing.
- Multi-Stop TSP Route Builder: advanced path planning for multi-destination deliveries using a greedy nearest-neighbour TSP solver.
- Real OSM Graph: Andheri-Kurla corridor loaded from OpenStreetMap via osmnx, cached offline after first run.

## Setup and Execution

1. Initial Setup: double-click `setup.bat` (Windows) or run `bash setup.sh` (Linux/Mac).
   - Creates a Python virtual environment (venv).
   - Installs backend requirements (fastapi, torch, networkx, osmnx, etc.).
   - Installs frontend Node dependencies.
2. Run the System: double-click `run.bat` (Windows) or run `bash run.sh` (Linux/Mac).
   - Backend WebSocket server starts on http://localhost:8000.
   - Frontend Vite dev server starts on http://localhost:5173.
3. Demo: open the URL in a browser and use the Control Panel to:
   - Toggle Baseline / AI mode.
   - Adjust the Prediction Horizon slider.
   - Press Run Demo Script to see the full event sequence.

## How It Works

### 1. Predictor - STGCN (2-layer Spatio-Temporal GCN)
- Layer 1 applies a graph convolution (spatial) weighted by the row-normalised adjacency matrix.
- Layer 2 applies a 1D temporal convolution (Conv1d) over a 5-step sliding time window.
- Output head produces per-node (predicted_cost, risk_score) for the chosen horizon.
- Weather features (rain_intensity, wind_factor, is_storm) are injected into the node feature matrix each tick.
- All weights initialised with Xavier uniform. Model exported as TorchScript (stgcn_edge.pt).
- Inference verified under 300 ms on CPU at startup.

### 2. Router - RiskAwareRouter
- Builds a cost graph from predicted edge weights using:
  cost(edge) = 0.50 * travel_time + 0.35 * risk_score + 0.15 * fuel_penalty
- Uses greedy nearest-neighbour TSP for multi-stop deliveries.
- For 5 or fewer stops, uses exhaustive permutation search to guarantee optimal ordering.
- route_confidence = 100 - (max_risk_on_path * 100), clamped 0-100.
- rejected_route_reason explains how many edges exceed 60% flood probability.

### 3. Baseline Routing
- Pure Dijkstra on the static base_weight (distance only). No predictive inputs.
- Serves as the reactive benchmark.

### 4. Weather Feed - Open-Meteo
- No API key or registration required.
- Fetches rain (mm), wind speed (m/s), and weather code for Mumbai every 5 minutes.
- Degrades gracefully to neutral values if the network is unreachable.
- When rain_intensity > 0.6, flood risk is automatically boosted on low-lying nodes.

### 5. Backend Loop (approx. 0.5 s per tick)
- Updates vehicle positions, fuel, and event state.
- Fetches live weather from the 5-minute cache.
- Calls STGCN with current horizon and weather context.
- Generates AI and baseline routes for each vehicle.
- Broadcasts a JSON payload via WebSocket containing:
  - vehicles array with routes, times, risk, confidence, and rejected-route explanations.
  - predicted_graph for heat-map rendering.
  - state (event, frozen flag, horizon, logs).
  - latency_ms (decision computation time).
  - graph_source ("osm" or "synthetic").
  - weather (current rain_intensity, wind_factor, is_storm).
